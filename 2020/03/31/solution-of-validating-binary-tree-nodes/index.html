Solution of Validating Binary Tree Nodes

Posted on 2020-03-31

LeetCode Problem: 1361. Validate Binary Tree Nodes. Use disjoint-set to solve this problem. With Code.

Problem Description

There are n nodes with indexes from 0 to n-1, which may consist of a binary tree. There are two arrays leftChild and rightChild with size n. If node i don't have left child, leftChild[i] == -1. Otherwise the left child of node i is node leftChild[i]. Right child is with the similar rule. Check if all nodes form exactly one valid binary tree.

Analysis

You can see the introduction of disjoint-set (also called Union/Find) in Wikipedia.

Disjoint-set has two operations:

  • find: Find which set one element belongs to.
  • merge/union: Merge two different sets into a new set.

Firstly we can ignore the direction of the edges: If a directed binary tree is invalid, there is at least one cycle in its corresponding undirected tree.

Undirected tree has an important property: Each two nodes must have a unique path. We can use disjoint-set to represent this relationship: For every pair, if one node have a unique path to connect the other, they are in the same set.

Initially every node is in different set. We do a traversal from 0 to n-1:

  • Check if node i and its left child have been connected (use find to check it). If yes, connect them (use merge/union operation in disjoint-set). If no, you couldn't connect them again. If you do that, there is a cycle in the graph. So you should return false when this situation happens and stop this algorithm.
  • Check if node i and its right child have been connected.

After all nodes are passed, you should check if every node is in the same set. It can be checked quickly if you use union_by_size strategy in the disjoint-set.
